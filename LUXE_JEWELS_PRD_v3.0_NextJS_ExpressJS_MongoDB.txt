Architecture Overview
Frontend (Client-Side):
	•	Next.js 14 App Router
	•	React Components (TypeScript)
	•	Tailwind CSS + shadcn/ui
	•	Axios or Fetch API to call Express Backend

Backend (Server-Side):
	•	Express.js REST API
	•	Node.js Runtime
	•	Mongoose for MongoDB ODM
	•	JWT Authentication
	•	CORS enabled for Next.js frontend

Database:
	•	MongoDB (NoSQL database)
	•	Mongoose schemas for data validation
	•	Collections: products, categories, orders, users, etc.

Image Storage:
	•	Cloudinary - for image uploads, CDN, transformations
	•	URLs stored in MongoDB

AUTHENTICATION FLOW
Admin Login
	•	Admin enters email & password on /admin/login
	•	Frontend sends POST /api/admin/login to Backend
	•	Backend verifies credentials against MongoDB
	•	Backend generates JWT token (expires in 8 hours)
	•	Frontend stores token in localStorage
	•	All subsequent requests include Authorization header with token
	•	Backend middleware verifies token on protected routes

Customer Authentication
	•	Public customers do NOT need to login
	•	Cart stored in browser localStorage
	•	At checkout, customer provides email (not login)
	•	Order is created without customer account

DEPLOYMENT GUIDE
Frontend Deployment (Vercel)
	•	Push code to GitHub
	•	Connect repository to Vercel
	•	Set environment variables (NEXT_PUBLIC_API_URL, etc.)
	•	Deploy automatically on push

Backend Deployment (Railway or Render)
	•	Push code to GitHub
	•	Connect to Railway/Render
	•	Add environment variables
	•	Deploy with npm start

Database (MongoDB Atlas)
	•	Create MongoDB Atlas account
	•	Create cluster
	•	Generate connection string
	•	Add to Backend MONGODB_URI
QUICK START (LOCAL DEVELOPMENT)
1. Clone Repository
git clone <repo>
cd luxe-jewels

2. Setup Frontend
cd luxe-jewels-frontend
npm install
# Create .env.local
NEXT_PUBLIC_API_URL=http://localhost:5000
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=xxx
npm run dev
# Runs on http://localhost:3000

3. Setup Backend
cd luxe-jewels-backend
npm install
# Create .env file with all variables
npm start
# Runs on http://localhost:5000

4. Setup MongoDB
	•	Local: mongod (if installed locally)
	•	Or MongoDB Atlas: add connection string to .env

5. Test API
GET http://localhost:5000/api/products
Should return array of products
